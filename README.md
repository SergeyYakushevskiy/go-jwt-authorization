# Реализация jwt-авторизации на Golang
### Техническое задание
Cделать сервис авторизации и аутентификации на jwt c регистрацией пользователя. Субд обязательно reindexer. 
Обмен данными должен происходить в формате json.  Сервис должен уметь заблокировать учётную запись или аннулировать конкретную сессию.
Подготовить через  Postman/Yaak (рекомендуем именно yaak)

### Архитектура ПО
Архитектура приближена к стандартам DOD / PDD

### Организация хранения данных
Предполагается, что в БД хранится минимальный набор данных о пользователе:
- фамилия;
- имя;
- отчество;
- логин;
- пароль.

Аутентификационные данные отделены от пользовательских. Следовательно, выделяются следующие структуры данных:

profile:
- id
- surname
- name
- patronymic

credentials:
- user_id
- login
- password_hash
- isBlocked

tokens:
- user_id
- access_token
- refresh_token
