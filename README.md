# Реализация jwt-авторизации на Golang
### Техническое задание
Cделать сервис авторизации и аутентификации на jwt c регистрацией пользователя. Субд обязательно reindexer. 
Обмен данными должен происходить в формате json.  Сервис должен уметь заблокировать учётную запись или аннулировать конкретную сессию.
Подготовить через  Postman/Yaak (рекомендуем именно yaak)

### Архитектура ПО
Архитектура приближена к стандартам DOD / PDD

### Организация хранения данных
Предполагается, что в БД хранится минимальный набор данных о пользователе:
- фамилия;
- имя;
- отчество;
- логин;
- пароль.

Аутентификационные данные отделены от пользовательских. Выделяются следующие структуры данных:

profile:
- id
- surname
- name
- patronymic
- created_at

credentials:
- id
- user_id
- login
- password_hash
- isBlocked
- updated_at

jwt:
- id
- user_id
- refresh_jti
- device
- is_revoked

jwt_blacklist:
- id
- access_jti

Для реализации модели авторизации пользователей были выделены следующие конечные маршруты:
  - "/api/register" - регистрация пользователя, выдача profile_id
  - "/api/login" - аутентификация и авторизация пользователя, выдача access- и refresh-токенов
  - "/api/refresh" - выдача нового access-токена по имеющемуся допустимому refresh-токену
  - "/api/profile" - маршрут ограниченного доступа для возвращения данных о пользователе
  - "/api/revoke" - отзыв токена по его id (jti)
  - "/api/logout" - завершение "сессии", отзыв токенов
  - "/api/sessions" - маршрут ограниченного доступа для просмотра всех активных "сессий"
